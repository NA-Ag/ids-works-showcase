IDS Master Document: Zero-Cost Educational Software Strategy

Business Model: "Pay Once, Own Forever" (Perpetual Licensing)
Target Market: Private Schools in Interlomas, State of Mexico, and nationwide.
Core Philosophy: Zero recurring costs for the developer (no paid APIs, no paid servers) and zero recurring costs for the schools. Undercut VC-backed SaaS platforms by offering robust, locally-hosted solutions.

Immediate Financial Goal & Ultimate Objective:
The immediate revenue target is $500,000 MXN. Any revenue beyond this is passive income. The primary purpose of this pivot is to create a "set it and forget it" automated business that generates cash flow without requiring manual developer intervention or maintenance. This absolute time-freedom will fund and allow complete focus on the true passion project: the Godot port of Penko Adventure (language learning app).

1. Technical Architecture & Distribution

Sales Platform (Frontend): GitHub Pages. Completely free hosting for the static idsworks.com.mx website.
Software Framework: Electron.js. Allows us to build cross-platform desktop applications (Windows, macOS, Linux) using web technologies (HTML, CSS, JavaScript) + SQLite for local databases.
Distribution: GitHub Releases. Download links point to the respective GitHub release assets (free bandwidth).

Local Networking (The "Cloud" Replacement): 
Applications designed for multiple users will utilize local network protocols (LAN) via Node.js (Express). 
*CRITICAL FIX:* Because school networks (VLANs/Firewalls) are messy, relying solely on auto-discovery (mDNS) will cause support tickets. The software MUST include a robust "Connection Wizard" that displays the Host's exact IPv4 address, allowing client computers to connect manually if auto-discovery fails.

Licensing System: Offline cryptographic hash keys validated locally. It allows exactly the specified number of unique local IP/MAC addresses to connect to the host.

2. Payment Automation & Fulfillment Strategies

To achieve true passive income, fulfillment must be bulletproof. 

Chosen Strategy: Serverless Webhooks + Mercado Pago (The "Sleep Well" Route)
*Why not email parsing?* Parsing bank emails (CoDi) is too fragile. If the bank changes a single pixel in their email template, the system breaks, schools don't get their software, and the "passive" income becomes an active customer service nightmare.
How it works:
1. Create a checkout link via Mercado Pago.
2. Mercado Pago sends a Webhook to a free Cloudflare Worker (100k free requests/day).
3. The Worker generates the cryptographic license key and uses a free email service (Resend) to email the key and download link to the school.
4. Cost: ~3.49% + $4 MXN per transaction, which is a cheap price to pay for 100% automated reliability.

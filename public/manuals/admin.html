Suite de Administración Escolar

Objetivo Principal

Construir un Sistema de Administración Escolar (SIS/ERP) robusto, con funcionamiento primordial fuera de línea (offline-first), centrado en el registro de alumnos, seguimiento de inscripciones, directorios de padres y generación de documentos internos.

Justificación de Inversión y Valor ($15,000 MXN)

El costo de $15,000 MXN por una licencia perpetua para 20 equipos es altamente competitivo frente a los cargos recurrentes de plataformas SaaS.

  • Los sistemas tradicionales de gestión escolar en la nube cobran tarifas mensuales por alumno o usuario, lo que rápidamente puede superar los $50,000+ MXN anuales para una escuela mediana.
  • Este módulo elimina completamente los costos recurrentes. Funciona en la red local (LAN) existente de la escuela sin requerir acceso a internet.
  • Provee soberanía y privacidad absoluta de datos, una preocupación creciente en las instituciones educativas de México.
  • Genera instantáneamente los documentos internos oficiales requeridos (ej. formatos de cumplimiento SEP, constancias de inscripción) sin depender de una conexión a internet inestable.

Arquitectura Técnica

  • Framework: Electron.js
  • Frontend: React 19, TypeScript, Tailwind CSS, Lucide React.
  • Servidor Local / Backend: Node.js (Express) ejecutándose en la máquina designada como "Host" para servir la API a las demás computadoras en la red LAN.
  • Base de Datos: SQLite (base de datos local basada en un archivo; no requiere instalación ni mantenimiento por parte del personal de TI de la escuela).
  • Plataformas Objetivo: Windows y Linux ÚNICAMENTE.
    Nota: macOS se excluye explícitamente para evitar las restricciones de certificación de Apple y costos recurrentes de licencias. Las administraciones escolares utilizan predominantemente Windows.

Investigación de Código Abierto y Alternativas

Antes de la construcción, analizamos soluciones de código abierto existentes para comprender la paridad de funciones y las estructuras de datos:

  1. Frappe Education (ERPNext): Basado en Python. Extremadamente completo pero muy pesado, requiere una configuración de servidor compleja (MariaDB, Redis). Demasiado complicado para una experiencia simple "instalar y usar" sin conexión.
  2. RosarioSIS: PHP/PostgreSQL. Ligero y popular para educación K-12, pero requiere instalar una pila de servidor web (LAMP/LEMP). Excelente referencia para esquema de datos (demografía, asistencia, calificaciones).
  3. Gibbon: PHP/MySQL. Altamente flexible con plugins, pero nuevamente depende de servidores web o acceso a internet.

Conclusión: Las herramientas de código abierto requieren conocimientos técnicos de TI para configurar servidores y bases de datos. Nuestro enfoque (Electron + SQLite + Node.js en LAN) ofrece el poder de una aplicación web con la instalación de cero configuración de un .exe de escritorio, brindándonos una ventaja competitiva masiva en facilidad de uso.

Características Clave (Ventaja Competitiva para México)

Para superar completamente a las alternativas SaaS de paga y de código abierto en el mercado mexicano, el software resuelve problemas burocráticos específicos y dolorosos que el software genérico ignora. Estas funciones hacen que el pago único sea una inversión evidente:

  • Validación Automática de CURP y RFC: Las herramientas genéricas permiten errores de dedo. Nuestro software incluye un algoritmo fuera de línea para validar instantáneamente si un CURP o RFC (de un tutor) es estructuralmente correcto antes de poder guardarse, previniendo rechazos posteriores de la SEP.
  • Descarga Administrativa (Exportación SEP): Actualmente las escuelas sufren la "doble captura" (escribir calificaciones en su sistema y luego otra vez en portales como SICEP o AEFCM). Mapeamos nuestra base de datos para generar exactamente los archivos .csv o .xml requeridos para carga masiva gubernamental.
  • Constancias con Código QR Local: La SEP tarda meses en emitir certificados oficiales. La Admin Suite genera "Constancias de Estudios" hermosas y listas para imprimir, que incluyen un código QR generado localmente que enlaza a un ID de validación, dando legitimidad institucional inmediata ante los padres.
  • Protección Contra Caídas de la SEP: Durante el fin de curso, los portales gubernamentales colapsan. Al ser 100% offline, el personal puede continuar trabajando, capturando calificaciones e imprimiendo boletas internas sin interrupción.

Características Esenciales (Módulos Base)

  1. Gestión de Alumnos: Agregar, editar y buscar perfiles de alumnos (demografía, alertas médicas, grupos asignados).
  2. Directorio de Padres/Tutores: Enlazar alumnos con perfiles de padres con información de contacto.
  3. Inscripciones / Grupos: Asignar alumnos a ciclos escolares, grados y grupos específicos (ej. "1A", "3B").
  4. Generación de Documentos: Crear constancias en PDF, listas de asistencia y cartas administrativas básicas.
  5. Host de Red Local: Un panel de configuración para iniciar el servidor local, habilitar la encriptación E2EE (AES-GCM) y mostrar la dirección IP local para que otros clientes puedan conectarse de forma segura.
